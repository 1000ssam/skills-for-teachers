## D3. (원칙) 결정론은 심판이 아니라 분류다 — 표면축 vs 의미축
**상태: 원칙 확정·능동 적용(SKILL §6에 반영). D1·D2가 이 원칙을 따른다.**

함정은 결정론 자체가 아니라 **의미 문제에 결정론이 성공/실패 판결을 내리게 두는 것**이다.
검증 신호를 두 축으로 가른다:

| 축 | 예 | 결정론 권한 |
|---|---|---|
| **표면축** (문자열이 곧 진실) | 바이트, 금지어 정확일치, 점수 패턴, 명사형 종결, 특수문자 | **하드 판정 OK** (문맥 없이 참) |
| **의미축** (문맥이 진실을 정함) | 근거 정박, 성취수준, **날조 여부** | **판결 금지 → advisory 힌트 / Tier-2 worklist** |

- verify.py의 성취수준 힌트가 advisory(헌법4)인 것이 이 원칙의 실현. 날조도 의미축 → Tier-1이 판결하지 않음(SKILL §6 "날조=자동수정 금지, 하드 플래그(사람)").
- **엣지 케이스 처리(의역/정규화)**: 학생 "임진년에 발생한 조선과 일본의 전쟁" → 초안 "임진왜란".
  - substring 하드 게이트라면 "원문에 없는 용어=날조" **오탐**.
  - 정정된 처리: Tier-1은 "verbatim 부재 용어"를 **Tier-2 worklist**로만 넘김. Tier-2/교사가 "의역·정규화(윤문=정당) vs 발명(날조)"을 판정.
  - 근거: 기재요령 AI조항이 윤문 보조수단 허용, 헌법2도 구어체→학술어 순화를 막지 않음(복원이지 창작 아님).
- **남는 한계(정직)**: density 프록시는 의역에 둔감(다 풀어 쓴 학생이 용어 박은 학생보다 앵커 수 낮게 잡힐 수 있음). 표면 프록시의 태생적 한계 → 그래서 힌트는 사용자에게 성공/실패로 안 보이고, 최종 판결은 Tier-2+교사 눈검수(헌법6).
- **교훈**: 앞서 "verify.py가 근거 화이트리스트를 결정론으로 강제"라 말한 것은 과한 표현이었고 정정됨. 결정론은 180명을 값싸게 **분류(triage)**해 사람 눈이 갈 곳을 좁힐 뿐, 의미의 심판관이 아니다.

## D1. (기능) Step 4.5 — 추출전용 '증거 ledger' JSON
**상태: 원칙 확정. 스키마는 첫 실전 OCR 실물 확보 후 확정(실물에서 역산).**

RAW(디지털) / 세미-RAW(OCR 보정본)을 Step5(작성) 전에 **구조화 JSON**으로 전환.
목적: 추출(무엇이 종이에 있나)과 작성(세특 문장)을 분리 → 배치 감사 가능 + 날조 가드의 기준점(ground truth) 확보.

**🚨 반드시 서술 필드만. 평가 필드 금지(날조 함정).**
- ✅ 서술적(종이에 뭐가 있나): `원문_스팬`(verbatim 인용), `언급_근거`(고유명·수치·사료·용어), `판독불가`, `활동주제`(spec.task에서), `과목`, `제출형태`.
- ❌ 평가적(잘했나): `역량`·`수행과정 품질`·`성취수준` — **빈 필드는 채우고 싶어져 LLM이 발명함**. 특히 `역량`은 학생 원문에 없는 것(=교사 렌즈)이라 추출 단계 금지. 이것들은 Step5가 렌즈로 붙이고 verify/Tier2가 이 ledger와 **대조**.

스케치(실물 확인 후 확정):
```json
{
  "학생식별": "3-14", "과목": "한국사", "활동주제": "임진왜란 용어 비판",
  "제출형태": "손글씨스캔",
  "원문_스팬": ["왜란은 일본 입장 뺀 거 같다", "임진전쟁이 더 중립적"],
  "언급_근거": ["임진전쟁", "왜란"],
  "판독불가": [], "비고": "구어체 다수"
}
```

**`언급_근거`의 역할**(왜 원문_스팬의 부분집합인데 따로 두나 = "검증·작성이 붙잡을 손잡이" 색인):
1. **L3(근거 정박) 결정론 계량 재료** — 비면 일반명사로 성긴 하(下) 신호, 차면 상 신호. Tier-2에만 있던 판단을 일부 Tier-1로 당김.
2. **작성자 '승격' 대상** — 새 근거 발명 금지, 이 목록의 앵커만 문장으로 끌어올림(창작 반경 고삐).
3. (❌ 날조 화이트리스트 하드 게이트로는 쓰지 않음 — D3. 의역이 오탐 유발. Tier-2 worklist 입력으로만.)

**철칙**: `언급_근거` 항목은 반드시 `원문_스팬`의 **verbatim 부분문자열**(의역·요약 금지). verify.py가 "모든 언급_근거가 원문에 실재하나"를 결정론으로 강제 가능 — 이건 *우리 추출물의 무결성* 검사라 표면축(하드 OK, D3).

**왜 지금 안 짓나**: 실제 학생 원문의 거칠기를 모르는 채 스키마 확정 = 헛다리. 추출 방법(정규식 고유명·수치만 vs LLM 추출 후 substring 검증)도 실물 보고 결정.

## D2. (기능) 교사 고유 역량 표현 — 스펙별 lens_lexicon 오버레이
**상태: 워크플로 훅은 이미 존재(eval_lens). Tier-1 오버레이는 첫 실전 스펙이 오탐 낼 때 구현.**

교사가 표준 대신 자기 역량 표현("사료 비판적 독해력" 등)을 쓰면, KICE 정박 범용 베이스라인(`recommended-structure.txt`)이 이를 천장으로 못 세워 "천장 부재→단순나열" 오탐 가능.

**워크플로 반영(이미 됨)**: `eval-context-spec`의 `eval_lens`(=평가요소)·`individuation_axes`가 교사별 프레이밍을 서술로 흡수(SKILL §1). Step1 인테이크가 "국가 성취기준 말고 당신의 평가철학으로"를 묻는 이유.

**Tier-1 동적 반영(구현 대기)**: 스펙이 교사 어휘를 실어 나르고 verify.py가 베이스라인+오버레이 병합.
```yaml
# eval-context-spec-한국사-2026.md 안에
lens_lexicon:
  ceiling.lens: [사료 비판적 독해력, 역사적 감정이입, 다원적 관점]
```
```bash
python3 verify.py drafts/ --lens eval-context-spec-한국사-2026.md
```
- verify.py 변경 = 사전 dict 하나 더 merge(≈5줄). 이미 파일 dict 로드 구조라 자연스러움.
- **가산만(additive)**: KICE 베이스라인 유지, 교사 표현 얹기 → 정박 손실 0.
- **출처 라벨 분리**: 오버레이 항목은 `[SPEC]` 태깅(≠`[KICE]`) → 감사 추적 정직.
- **천장에만 허용**: 극성·자율성은 과목·교사 불변(구조적)이라 오버레이 금지.
- "동적" = 베이스라인 안 건드리고 활성 스펙에서 로드 = 배치마다 자동 교체.

**왜 지금 안 짓나**: 교사가 실제로 비표준 어휘를 쓰는지 미확인. 훅(eval_lens)은 이미 있으니 실제 오탐 발생 시 붙임.
