# Student Record Pipeline (세특 엔드투엔드 파이프라인)

학생 산출물(손글씨 활동지 스캔 · 사진 · 디지털 텍스트)을 받아
**교과세특(과목별 세부능력 및 특기사항)** 초안 배치를 만드는 파이프라인.

`student-record-writer`(단건 작성기)를 계승하되, 그 앞뒤로
**인테이크 QA → 인제스천 → 매핑 → 맥락 보정**을, 뒤로 **바이트 예산·검증**을 붙였다.

> ⚠️ **기준 학년도 2026.** 기재요령은 매년 개정된다 — 새 학년도엔 `references/giwan-*-grounding.md`·바이트 규칙·`forbidden-terms.txt`를 갱신하고 `subject` 스펙에 기준 학년도를 명시할 것.

## 0. 헌법 (INVARIANTS) — 절대 어기지 않는다

이 6개는 파이프라인의 불변식이다. 어느 단계에서도 위반 금지.
**전 조항은 `references/giwan-2026-grounding.md`(2026 기재요령 발췌, 경기 goe.go.kr)에 근거.**

1. **매핑은 조용히 추측하지 않는다.**
   명렬표 정본이 유일한 ground truth. 파일 순서(위치 prior) + OCR이 읽은 이름/학번(노이즈 신호)을
   정본과 **3중 대조**해, 셋이 일치할 때만 자동 확정. 하나라도 어긋나면 ⚠️ 플래그 → 사람 검수.
   **오매핑(엉뚱한 학생 세특) 0이 최우선.** 인간 노동이 늘더라도 이 원칙이 이긴다.
   🗺️ **단, 헤더 문자열 불일치 ≠ 오매핑.** 손글씨 OCR 오독이 흔하다(본인 이름 1~2글자 오독, 학번 1자리 오독, **학번/이름 칸 바꿔쓰기**). 헤더가 배정값과 달라 보일 때는 두 신호로 오독 vs 진짜 오매핑을 가른다:
   ① **충돌값이 명렬표에 실재하는 다른 학생인가?** 실재조차 안 하면(예: 명렬표에 없는 이름) = 본인 이름 OCR 오독일 뿐.
   ② **한 학생의 여러 낱장(p3·p4 등) 본문이 동일 서사(같은 사건·인물·시점)로 이어지는가?** 이어지면 같은 학생의 한 제출물.
   → ①이 '실재 안 함' + ②가 '일관' 이면 **오매핑 아님**(형식적 스캔 사인만). 충돌값이 실재 타 학생이고 그 내용이 섞였거나 낱장들이 서로 다른 이야기일 때만 진짜 오매핑으로 hold. **결정적 방어선: 오매핑은 반드시 헤더 불일치를 남기므로, 헤더불일치 hold 전수를 위 두 신호로 대조하면 오매핑 0을 증명할 수 있다.**

2. **보정은 복원까지만. 보강·창작 금지 (🚨 no-fabrication).**
   맥락 보정은 OCR로 깨진 글자를 *올바로 읽는 사전(辭典)* 역할일 뿐이다.
   학생이 실제로 쓴 것을 복구할 뿐, **학생이 안 한 성취를 채워 넣지 않는다.**
   읽을 수 없는 구간은 지어내지 말고 `[판독불가]`로 플래그.
   근거: 허위·과장 기재는 '학생성적 관련 비위'로 「교육공무원 징계양정 규칙」 적용(기재요령 p.24~25, 4-가/다).

3. **성취기준·성취수준은 2층으로 쓴다.**
   - 근거층: 국가 교육과정 성취기준 → **문장에 어휘로 드러나지 않는다.** 감사·이의제기 대비 뒷받침용.
   - 렌즈층: **교사 평가 렌즈 = 공식 용어 '평가 요소'**(성취기준 도달의 증거로 학생이 보여주길 기대한 핵심 내용, p.98)
     → 문장을 실제로 이끄는 주체. 개별성의 원천.
   근거(p.94): 세특은 성취기준·성취수준에 **근거**하되 **"수업 활동의 단순 나열이나 성취기준에 명시된 지식의 단순 서술은 지양"**.
   국가 성취기준으로 문장을 몰면 곧 이 '단순 서술'이 되어 플랫해진다. 렌즈가 표면, 성취기준은 침묵.

4. **바이트 예산은 상한이지 하한이 아니다.**
   목표 범위(예: 500~650 byte)는 "잘 쓴 학생의 착지 구간"일 뿐.
   산출물이 빈약하면 그 밑으로 내려가도 정상. **분량 채우려는 padding = 2번 위반.**

5. **점수·등급·석차를 본문에 직접 쓰지 않는다.**
   "홀리스틱 14점", "성취수준 B", "상위 몇 %" 금지(기재요령).
   루브릭 점수/수준은 문장의 깊이·톤을 정하는 **내부 신호로만** 쓰고 서술로 녹인다.

6. **출력은 언제나 '초안 + 상태'. 교사 눈검수가 NEIS 입력 전 필수 관문.**
   파이프라인은 최종본을 만들지 않는다. 생기부는 법적 문서이고 최종 책임은 교사에게 있다.
   근거(p.25, 4-다): 기재요령은 "**AI 생성물을 서술형 항목에 그대로 입력하는 행위**"를 금지하되,
   "**윤문 등 보조 수단**으로 AI를 쓸 경우 **최종 입력 전 허위·과장 여부·유의사항 준수를 철저히 확인**"하면 허용.
   이 스킬은 그 **검증을 강제하는 보조수단**으로만 존재한다.

## 1. 평가 맥락 스펙 (매번 갈아끼우는 단위)

인테이크 QA가 `references/eval-context-spec.template.md`(빈 템플릿) 형태로 수집한다.
이 스펙 하나가 파이프라인 전체를 구동한다. 과목·학기·과제가 바뀌면 이 파일만 교체.

> 🔒 **데이터 경계(절대):** 스킬 폴더(`references/`)에는 **범용 자산만**(빈 템플릿·사전·verify·헌법). **채워진 스펙·명렬표·OCR/전사 원문·증거 ledger·세특 초안 = PII → 스킬 밖 프로젝트 작업폴더**(`<project>/`, 스킬과 분리)에 둔다. 이 스킬 폴더는 버전관리·공유 대상이므로 **학생 실명·산출물을 절대 커밋하지 않는다.**

| 필드 | 내용 | 성격 |
|---|---|---|
| `subject` | 과목·학기·대상 학년/반 | 식별 |
| `task` | 과제 정체 — 무엇을 시켰나(지시문/조건) | **필수** |
| `artifact_form` | 산출물 형태 → 입력 어댑터 결정 (손글씨스캔 / 사진 / 디지털텍스트) | 필수 |
| `rubric` | 채점 척도 + 점수대별 판정 (예: 홀리스틱 8~20, 2점 단위) | **필수** |
| `eval_lens` | **평가 렌즈 = 평가 요소**(공식용어, p.98) — 이 과제로 진짜 보려던 것(교사 수준 교육과정) | **필수·개별성 원천** |
| `standards_backing` | 국가 성취기준 코드 (`[10한사…]`/`[12동사…]`) | 선택·침묵 근거 |
| `roster` | 명렬표 정본(번호·이름, 가능하면 학번) 파일 경로. **NEIS 과목세특 내보내기 xlsx면 최상** — 명렬표 겸 최종 이관 타깃(Step 6 write-back) | 매핑 ground truth |
| `length_budget` | 목표 바이트 범위 + 바이트 규칙(기본 NEIS) | **인테이크에서 질문** |
| `individuation_axes` | 학생마다 갈리는 지점(무엇을 근거로 A와 B를 다르게 쓰나) | 플랫 방지 |
| `extra_forbidden` | 이 과제 특유의 주의/금지 | 선택 |
| `model_config` | 작성/검증/재작성 3단 모델 조합 — 규모·품질민감도로 결정하고 **예상 토큰·시간·비용 견적 확인 후 확정**. 기본값=작성 Sonnet-med·**검증 Sonnet-high(핵심 레버)**·재작성 Opus (접근·비용 여건 따라 한 단계씩 하향 가능) | **인테이크에서 질문** |
| `style_profile` | 교사 문체 프로파일 — 교사 예시 3~5문장에서 **문체만 추출**(문장 길이·개수, 선호 종결어미, 자주 쓰는 역량표현 관용구, 접속 습관, 선호/기피 어휘). ⚠️ 예시 원문장은 타 학생 PII → **로컬(교사별)만 보관**, 프로파일만 WRITE_PREFIX에 주입 | 선택·문체 앵커 |
| `overflow_policy` | 바이트 상한 초과 시 처리: 자동 트림(≤목표) vs 소폭 허용(허용 상한) | 인테이크에서 질문 |

## 2. 파이프라인 6단계 (+ Step 7 NEIS 이관, 서식 확보 시)

> 📁 `<스킬경로>` = 이 스킬이 설치된 폴더(예: `~/.claude/skills/student-record-pipeline`). 아래 스크립트 실행 시 실제 설치 경로로 치환.

### Step 1 — 인테이크 QA (기본 진입점)
새 세특 작업을 시작하면 **먼저 교사를 인터뷰**해 §1 스펙을 채운다.
아래 질문 세트를 1문1답으로 진행하고, 각 질문에 **추천 답**을 제시한다.
스펙이 완성되면 **프로젝트 작업폴더**(`<project>/`, 스킬 밖)에 `eval-context-<subject>-<date>.md`로 저장(§1 데이터 경계 🔒).

인테이크 질문 세트:
1. 과목·학기·대상은? 대상 인원 대략 몇 명?
2. **과제**: 학생에게 무엇을, 어떤 조건으로 시켰나? (지시문 원문이 있으면 그대로)
3. 산출물 형태? (디지털 텍스트 → 직결 / 스캔·사진 → 임의 OCR 또는 담당 에이전트의 비전 축자전사)
4. **루브릭**: 채점 척도와 점수대별 판정 기준?
5. **평가 렌즈**: 이 과제로 *진짜* 보려던 건 무엇인가? 국가 성취기준 말고, 당신의 평가 철학·평가요소로.
6. **분량**: 학생당 목표 바이트 범위? (기본 제안 500~650 byte ≈ 한글 165~215자, 상한 취급)
7. 명렬표 정본 파일 경로? (번호·이름·학번)
   💡 **NEIS 과목세특 내보내기 xlsx가 있으면 그걸 주세요(최상).** 이 서식은 (a) 명렬표 정본(반/번호·성명·학번)이자 (b) 완성본을 그대로 다시 업로드하는 **최종 이관 타깃**을 겸하므로, Step 6에서 결정론 write-back(`neis_writeback.py`)이 가능해집니다. NEIS `학생부 > 과목별 세부능력 및 특기사항 > 엑셀 내려받기`로 반별 파일을 받으면 됩니다.
8. 개별화 축: 학생 A와 B를 다르게 만드는 근거는? (렌즈에서 보통 파생)
9. 국가 성취기준 코드(선택), 과제 특유 금지사항(선택).
10. **모델·예산**: 규모(N명)와 품질 민감도는? → 이걸로 작성/검증/재작성 3단 모델 조합을 **추천**하고 **예상 토큰·시간·비용 견적을 먼저 제시**한 뒤 확정한다(`model_config`). 기본 균형값=작성 Sonnet-med·검증 Sonnet-high·재작성 Opus. **검증관 모델이 품질 핵심 레버**(적대적 판정).
11. **문체 앵커(선택)**: 선생님이 **직접 쓴 세특 예시 3~5문장** 있으면 주세요 → 거기서 **문체 프로파일만 추출**(문장 길이·종결·역량표현 관용구·선호/기피 어휘)해 작성에 반영합니다. ⚠️ 예시엔 과거 학생 PII가 있으니 **원문장은 로컬(교사별)만 보관**, 프로파일만 씁니다(공유 스킬에 원문 저장 금지). 없으면 스킬 기본 문체로 진행.
12. **초과 정책**: 바이트 상한을 넘으면 자동으로 줄일까요(트림), 소폭 허용할까요? (`overflow_policy`)

### Step 2 — 인제스천 (입력 어댑터)
`artifact_form`에 따라 분기. **핵심 계약 = "학생 산출물 원문 텍스트"**(어떻게 얻었든 무방). 하류(매핑~검증)는 텍스트만 받으므로 어댑터는 교체 가능하다. **특정 OCR 벤더에 종속되지 않는다.**
- **① 디지털 텍스트**(구글폼 응답·한글/워드 파일·타이핑) → 그대로 사용(직결).
- **② 스캔/사진 + 외부 OCR 보유** → 임의 OCR로 텍스트 추출(클로바·구글·네이버 등 무엇이든 — 예시일 뿐). 파일별 `raw/*.txt` 저장, 파일 순서 보존.
- **③ OCR 없음 → 담당 에이전트의 비전(멀티모달) 능력으로 축자(verbatim) 전사.** 거친 손글씨는 비전이 전용 OCR보다 나을 때가 많다. **단, 반드시 "복사기" 모드로 강제** — 비전 모델은 방치하면 오타를 고치고 반쯤 쓴 말을 완성해 **원자료를 조용히 날조**함(헌법 2 위반):
  - 전사 프롬프트 가드: *"보이는 글자를 그대로 옮겨라. 맞춤법·띄어쓰기·문법 **교정 금지**. 못 읽는 글자는 `[판독불가]`. 빈칸은 빈칸으로, 비문은 비문대로."*
  - 산출도 `raw/*.txt`로 남겨 보정·검증·교사검수 관문이 동일하게 걸리게 함.
  - **비전 출처 플래그:** 이 배치는 원자료가 비전이므로 verify·교사검수에서 날조 경계치를 한 단계 올림.
  - **매핑 독립성 주의(헌법 1):** 비전이 읽은 이름/학번은 *독립적 3번째 신호가 아님*(기대값 확증편향). 3중 대조에서 위치-prior·명렬표를 더 무겁게 잡고 ⚠️는 교사 확인으로.
  - **규모 단서:** 한 반 규모는 비전이 편하나, 대량·다페이지는 토큰·속도·컨텍스트 오염이 커지니 전용 OCR 권장.
- 셋 다 **동일한 하류 플로우로 합류.**
산출: 학생 후보별 `{파일/식별, 원문텍스트, (있으면) 이름·학번}` 레코드 목록.

### Step 3 — 매핑 (헌법 1)
명렬표 정본을 로드. 각 정본 학생에 대해:
- 위치 prior(파일 순서) + OCR 이름/학번 대조.
- 3자 일치 → ✅ 자동 확정.
- 불일치/모호/미제출/오독 → 상태값 부여, **절대 추측 확정 금지.**
산출: **검수 테이블** — `정본번호 | 이름 | 매칭파일 | OCR읽은값 | 상태(✅일치/⚠️불일치/❌미제출/❓오독)`.
⚠️/❓/❌는 교사에게 넘겨 해소한 뒤에만 다음 단계 진입.
최악의 경우 교사가 파일명을 직접 달아 주는 경로도 허용(그래도 정본 대조는 수행).

### Step 4 — 맥락 보정 (헌법 2)
확정된 학생별 원문에 대해, §1 스펙(과제·루브릭·평가렌즈·과목지식)을 **디코더**로 써서
OCR 노이즈(오탈자·띄어쓰기 붕괴·오독·잘림)를 교정한다.
- 문맥상 명백한 오독만 복원(예: `의벙`→`의병`).
- 뜻이 불확실하면 복원하지 말고 `[판독불가]` 유지.
- **내용을 더하지 않는다.** 산출물이 얇으면 얇은 대로 다음 단계로.
디지털 텍스트 입력은 이 단계 사실상 no-op(경미한 정규화만).

### Step 5 — 세특 작성 (student-record-writer 4단계 계승·확장)
학생별 보정 원문 + 스펙으로 세특 초안 작성. 규칙:
- **구조(4단계 = 권장 구조, `references/recommended-structure.txt`):** ①성취수준(도입) → ②수행 과정·**구체적 근거**(반드시 포함)
  → ③역량·성취특성(개별성) → ④종합·성장·후속. **단순 나열·성취기준 지식의 단순 서술 지양(p.101).**
- **렌즈 주도:** 문장은 `eval_lens`가 이끈다. `standards_backing`은 침묵 근거(헌법 3).
- **성취수준 앵커:** ①도입은 루브릭 판정에 근거한 *실제 도달 수준*을 서술(점수 라벨 금지, 헌법 5).
- **바이트 예산:** `length_budget` 상한 준수. 근거 없으면 짧게(헌법 4).
- **문체:** '~함/~임/~보임' 명사형 종결(p.36). **특수문자·문단구분 기호(번호)·가운뎃점(·) 지양(p.36).** 나열은 가운뎃점으로 압축하지 말고 산문으로 풀어 씀.
  구어체는 학술어로 순화하되 학생 고유 비유는 인용해 살림.
- **도서 인용(일관·재현·재확인):** 도서를 언급하면 **배치 전체에서 한 형식으로 일관**되게 쓴다. 기본 `'제목(저자)'`(교사가 `style_profile.citation_format` 지정 시 그에 따름 — 작은따옴표·괄호는 기재요령 '특수문자 지양'과 저촉될 수 있어 학교별 판단). ⚠️ 제목·저자는 **학생 산출물에 쓰인 대로 재현**한다(헌법 2 — canonical 서지를 리서치·교정해 채우는 순간 그게 발명·오염). 기계가 채운 서지는 학생 오기 포함해 틀릴 수 있어 **정확성 보증 불가 → Step 6에서 교사 재확인으로 넘김**(아래 산출).
- **가독성·만연 방지(문장 재구성):** 한 문장에 절을 계속 이어붙여 100자를 넘기지 않음. **목표는 "짧게 자르기"가 아니라 "복문을 읽기 좋은 리듬으로 재구성"** — 글자 수는 결과 지표일 뿐(`sentence_metrics.py`가 후보만 지목).
  - ① **논리 관절에서만 끊음**(근거→물음→결론). 의미로 묶인 절(병렬 물음 등)은 한 문장에 유지 — 절 단위로 잘게 부수면 뚝뚝 끊긴 덩어리가 됨.
  - ② **문두 접속 표현으로 관계 복원**(이를 실마리 삼아 / 그 물음 끝에 / 나아가). 접속 없이 마침표로만 끊으면 관계가 증발.
  - ③ 쪼갠 각 문장도 **명사형 종결 유지하되 종결어미 다양화**(~함/~임/~보임/~드러냄/~이름/~에 이름 로테이션)해 단조 방지.
  - ④ **쉼표 절제:** 관행상 생략 가능한 쉼표(문두 접속어 뒤 등)는 뺌 — 쉼표 범벅은 알려진 AI slop 신호. 단 **리스트·병렬 경계 쉼표는 유지**(어느 쉼표를 뺄지는 의미축이라 결정론이 못 정함, 작성자가 판단).
- **주술 관계(생기부 주어 = 학생):** 세특의 (암묵) 주어는 **항상 학생**. 문장 최종 서술어는 학생의 인지·수행 동사(논술함·분석함·논증함·재구성함·규명함·밝힘·파악함·짚음·서술함)여야 함. **개념·사건·사상 같은 무생물을 최종 서술어의 주어로 삼지 않음** — 개념은 학생 행위의 **목적어/내용**으로 넣음.
  - **재서술 2형(실전 검증):** 검출기는 **문장 최종 서술어 어절만** 보므로, 자체전개 동사를 최종 위치에서 몰아내면 플래그가 풀린다.
    - **형A · 목적어화** — 개념이 문두 주어일 때: 그 개념을 목적어로 돌리고 학생 동사로 닫음. `✗ "중화 개념이 …폄칭에서 출발함" → ✓ "중화 개념을 …폄칭에서 출발한 것으로 서술함"`(또는 "…출발한 개념으로 논술함").
    - **형B · 내포절 '…했음을'** — 앞에 살리고 싶은 부사구(…세계관에서/…더해져)가 있을 때: 절 전체를 `"…했음을 [학생동사]"`로 감쌈. 자체전개 동사는 비종결 위치로 내려가 그대로 보존되고, 어절 `"…을 밝힘/파악함"`만 붙어 **바이트가 싸다**(상한 근접 학생에 유리). `✗ "…세계관에서 중화 개념이 출발함" → ✓ "…세계관에서 중화 개념이 출발했음을 밝힘"` · `✗ "…우월의식이 더해져 중화가 성립함" → ✓ "…성립했음을 파악함"`.
    - 학생 동사는 문장마다 로테이션(서술함/밝힘/파악함/논증함/짚음/규명함)해 명사형 단조를 피함(§가독성 ③과 동일 원칙).
  - ⚠️ **만연 분할이 이 결함을 유입한다:** "…출발하고 …정리함"(학생=주어)을 관절에서 끊으면 앞 조각이 "…출발함"(개념=주어)이 됨. **분할 후 각 조각의 최종 서술어가 학생 행위인지 반드시 재확인.** `sentence_metrics.py`가 '-됨'류(형성됨·강화됨)·자체전개 자동사(출발함·성립함·굳어짐) 종결을 advisory로 지목(재서술은 LLM).
- **문체 프로파일 주입(선택, `style_profile`):** 교사 예시 문장이 있으면, 원문장을 few-shot으로 넣지 말고(내용 오염 방지) **추출한 프로파일**(문장 길이·개수, 선호 종결어미, 자주 쓰는 역량표현 관용구, 접속 습관, 선호/기피 어휘)만 WRITE_PREFIX에 규칙으로 주입한다. "이 교사는 보통 N문장·평균 K자, '~을 드러냄'류 종결 선호, 'A' 어휘 기피" 식. **원문장은 로컬 보관(교사별), 공유 스킬·프로젝트 산출물에 커밋 금지**(§1 🔒).
- **금지:** §금지사항 목록 전부. 단순 줄거리 요약 금지 → '변화·후속활동' 중심.
- 출력 본문에는 메타코멘트·제목·헤딩·라벨을 넣지 않는다(세특 문장만).

### Step 6 — 검증 루프 & 출력 (2계층)

> **원칙 — 결정론은 심판이 아니라 분류다 (표면축 vs 의미축).** 상세: `DESIGN-DECISIONS.md` D3.
> - **표면축**(바이트·금지어 정확일치·점수패턴·명사형종결·특수문자)은 문맥 없이 참이라 **하드 판정** OK.
> - **의미축**(근거 정박·성취수준·**날조 여부**)은 문맥이 진실을 정한다 → 결정론은 **판결하지 않고 advisory 힌트/Tier-2 worklist로만** 넘긴다.
> - 예: 학생이 "임진년의 조선·일본 전쟁"으로 **풀어 쓰고** 초안이 "임진왜란"으로 정규화하면, 문자열 부재를 날조로 하드판정하면 **오탐**이다.
>   Tier-1은 "verbatim 부재 용어"를 Tier-2 확인거리로만 전달하고, **의역·정규화(윤문=정당) vs 발명(날조)** 판정은 Tier-2·교사가 한다.
>   (근거 대조를 결정론 하드 게이트로 두지 않는다. substring 검사는 *우리 추출물의 무결성*에만 쓴다 — D1.)

**Tier 1 — 결정론 자동 (전수, `references/verify.py`)** — LLM 추정 금지, 스크립트로.
```bash
python3 <스킬경로>/references/verify.py <drafts_dir> --max <예산> --names <명렬표>
```
점검(위반=하드플래그): ①바이트/글자수(공통과목=1·2 합산 500자 캡, 예: 한국사) ②금지표현(`forbidden-terms.txt`) ③점수·등급·석차
패턴 ④명사형 종결 위반 ⑤특수문자·번호기호·**가운뎃점(·류 6종)**(세특 산문에서 나열 압축=AI slop, 하드플래그) ⑥`[판독불가]` 잔존 ⑦실명/타인명 삽입 ⑧영문 과다 ⑨개별성(배치 유사도).
**문장 만연도 표면 신호(advisory, `sentence_metrics.py`)**: ⑪ 문장 자수(경고≥90·목표≤80)·연결신호 밀도(≥4)·쉼표 범벅·문두 접속어+쉼표(안전 제거 후보)·문단 요약을 낸다. **만연 후보만 지목, 재구성은 LLM 몫**(어디서 끊고 어떻게 이을지는 의미축). 자수는 결과 지표이지 하드 컷 아님(헌법4).
**성취수준 다신호 추정(positive, advisory)**: ⑩ `recommended-structure.txt`(5층 어휘사전, KICE·교육과정 정박)로 **상/중/하 힌트**를 낸다 — L1천장(능력명사·질부사·칭찬클로저) + L2극성(완료형 vs 처방형 문말·헤지) + L4자율성(스스로 vs 지원발판). 부산물: **단순나열 의심**(바닥만·천장부재), **신호상충**(상↔하 혼재), `--levels`로 루브릭 등급 주면 **인플레/디플레**. **🔑 중립 태도어(참여·흥미)만으로는 태도서사로 안 보고, 진단 태도어(성실·극복·최선·활력소)일 때만 '하의 정당한 서술'로 인정** — 능력승화 부재를 단순나열로 오탐하지 않기 위함. **전부 경고일 뿐 자동 재작성 트리거 아님**(헌법4/2).

**Tier 2 — LLM 심판 (판단, 적대적 검증 권장)** — 결정론으로 못 잡는 것. Tier-1 힌트(⑩)를 입력으로 받아 **판정**한다.
- **★날조 가드(헌법2):** 초안의 각 성취 주장 ↔ 보정 원문 **근거 대조.** 근거 없는 문장 = ⚠️. "반증 시도" 프롬프트로.
- **★근거 정박 판정(L3, p.101 '구체적'):** 각 성취 주장이 **고유명·1차사료·수치·작품명·기법·인과사슬** 중 하나에 밀착했나, 아니면 일반명사로 성긴가.
  근거 밀도는 상>중>하로 갈린다(역사 계량 상6.3>하2.8). **정박 없는 칭찬(질부사·클로저만)은 공허** → 근거를 붙이거나 톤을 낮춘다. (없는 근거를 지어내는 건 금지 — 헌법2)
- **★능력승화 vs 태도승화 구별(SYN 보상천장·헌법2):** 초안이 천장으로 올라갔다면 그게 **능력·역량 승화(상: "분석력을 갖춤")**인가 **태도·성장 승화(하: "끝까지 최선을 다함")**인가.
  - 하 학생을 능력승화로 **부풀리면 날조**(⚠️ 하드). 하는 하답게 **태도·성장 서사로 정당하게** 쓴다(KICE "낮은 학생도 부정 진술 지양, 성취한 역량 중심").
  - 반대로 능력승화가 실재하는데 태도어로만 눌러쓰면 **디플레**. Tier-1 '신호상충'·'태도서사' 힌트가 뜬 초안을 우선 심판.
- **권장구조·평가요소 판정(p.101):** 렌즈(평가요소)를 반영했나, **권장 4요소(도달·구체적 근거·역량·성장)**가 서는가,
  성취기준 **단순 나열·단순 서술**로 흘렀나. (단, 산출물이 실제 빈약하면 짧은 게 정상 — 억지 채움 요구 금지)
- **성취수준 앵커 적정성:** 도입부 톤이 실제 루브릭 점수와 일치하나. Tier-1 `--levels` **인플레/디플레** 플래그를 근거 대조로 확증·기각.

**루프 동작:**
- 결정론 위반 중 **안전한 것**(바이트초과·명사형·특수문자·번호기호)만 위반내용을 되먹여 **재작성 후 재검증**(최대 N회). 바이트는 결정론 트림 대신 재작성(의미손실 방지).
- **만연 후보(advisory)도 안전 재작성 대상** — 단 **분할·재구성은 LLM이 수행**(관절 분할+문두 접속 복원+명사형 종결 다양화, 의미·근거·고유명 불변). 결정론은 후보만 지목(D3). Tier2 날조가드로 재구성 후 의미 불변 재확인.
- **가운뎃점(·)은 하드플래그이나 자동 치환 금지** — 단순 `·`→쉼표가 아니라 **나열을 산문으로 풀어 쓰는 재작성**(LLM/사람).
- **금지표현·성적표현·날조·실명 = 자동수정 금지 → 하드 플래그(사람).** (억지 수정이 곧 2차 날조)

산출: **교사 검수 테이블** — `번호 | 이름 | 세특초안 | 바이트 | Tier1플래그 | Tier2판정`.
- **📚 도서 인용 재확인(전수 게이트):** 초안에서 도서 인용의 `제목·저자`를 결정론으로 추출해 검수 테이블에 **'실제 책과 대조 요망'** 항목으로 함께 낸다. 서지는 기계·학생 오기로 틀릴 수 있으므로(canonical 리서치 금지=헌법 2) **NEIS 입력 전 교사 팩트체크를 강제**(헌법 6). 인용 없으면 생략.
교사 눈검수(헌법 6) 후 NEIS 입력 — 아래 두 경로 중 하나:
- **모드 A(수기):** 교사가 검수 테이블을 보고 NEIS에 직접 붙여넣기.
- **모드 B(결정론 이관):** NEIS 과목세특 내보내기 xlsx가 있으면 `neis_writeback.py`로 자동 write-back(아래 Step 7).

### Step 7 — NEIS 이관 (write-back, 모드 B) — NEIS 서식 확보 시
확정 세특을 **NEIS '과목별 세부능력 및 특기사항' 내보내기 xlsx**의 세특 열에 결정론적으로 써넣는다. 이 파일은 그대로 NEIS에 **재업로드 가능한 공식 서식**이므로, 붙여넣기 없이 반별 파일 하나로 업로드가 끝난다.

**서식 스키마(2026 고등, 자동감지 — 열 위치가 아니라 헤더명으로 매칭):**
- 시트 1개, **1행=헤더, 2행부터 데이터**, 반별 1파일(`…_과목세특_4.xlsx`, `_5`, …).
- 열: `학년도 | 학기 | 학년 | 학생개인번호 | 과목 | 과목코드 | 반/번호("4/1") | 성명 | 학적변동구분 | 세부능력 및 특기사항(=기입 타깃) | 영재·발명교육`.
- 🚩 이 서식의 세특 칸은 **과목 1개분**(예: 한국사1)만 담는다. 공통과목이면 합산 500자 캡(§3)을 1·2 **양쪽 서식을 합쳐** 관리 — write-back 전에 합산 바이트를 `neis_bytes.py`로 확인.

**이관 규칙(헌법 1을 write-back 경계까지 연장 + 헌법 6 관문 보호):**
- **이중키 대조:** 소스↔서식을 **(반, 번호) 복합키 + 성명 둘 다** 일치할 때만 기입. 학번 5자리 `10401 → 반04·번01` 파싱. 이름 불일치는 오매핑 위험 → **절대 안 씀**(플래그만).
- **non-empty 충돌 가드:** 타깃 세특 칸에 소스와 *다른* 기존 내용이 있으면 기본은 **전체 중단(halt) + 표면화**. 맹목적 덮어쓰기 금지 — 교사가 손본 최신본을 파괴할 수 있다(헌법 6). 검토 후 `--on-conflict skip`(기존유지)/`overwrite`(교체) 명시.
- **백업 우선 → 라운드트립 검증:** 쓰기 전 원본 전량 백업, 쓴 뒤 모든 셀을 다시 읽어 소스와 글자 대조.
- **양방향 대사:** 소스에만/타깃에만 있는 학생을 리포트(누락·초과 0 확인).

```bash
# 1) 대사만(아무것도 안 씀)  2) --write 로 실제 기입(충돌 시 자동 halt)
python3 <스킬경로>/references/neis_writeback.py \
  --source <확정세특.csv|json> --targets "<…_과목세특_*.xlsx>" [--write] [--on-conflict skip]
```
소스 CSV 기본 스키마 = `학번,이름,세특`(+선택 `상태`→`--status-col 상태 --status-ok 확정`). 반/번호를 별도 열로 주려면 `--class-col/--num-col`. 산출: 대사 리포트 + (기입 시) 백업경로·라운드트립 결과. **여기까지도 최종본 아님 — NEIS 업로드 전 교사가 서식을 한 번 더 눈으로 확인(헌법 6).**
